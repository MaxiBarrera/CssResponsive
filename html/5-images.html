Diseño web responsivo - Imágenes

Cambie el tamaño de la ventana del navegador para ver cómo se adapta la imagen a la página.

Uso de la propiedad width

Si la propiedad width se establece en un porcentaje y la propiedad height se establece en "automático", la imagen responderá y se ampliará o reducirá:

Ejemplo

img {
  width: 100%;
  height: auto;
}
Inténtalo tú mismo

Tenga en cuenta que en el ejemplo anterior, la imagen se puede ampliar para que sea más grande que su tamaño original. En muchos casos, una mejor solución será utilizar la propiedad max-width en su lugar.

Uso de la propiedad max-width

Si la propiedad max-width se establece al 100%, la imagen se reducirá si es necesario, pero nunca se ampliará hasta alcanzar un tamaño mayor que su tamaño original:

Ejemplo

img {
  max-width: 100%;
  height: auto;
}

Imágenes de fondo

Las imágenes de fondo también pueden responder al cambio de tamaño y escala.

Aquí mostraremos tres métodos diferentes:

1. Si la propiedad background-size está configurada como "contain", la imagen de fondo se escalará e intentará ajustarse al área de contenido. Sin embargo, la imagen mantendrá su relación de aspecto (la relación proporcional entre el ancho y la altura de la imagen):

Aquí está el código CSS:

Ejemplo

div {
  width: 100%;
  height: 400px;
  background-image: url('img_flowers.jpg');
  background-repeat: no-repeat;
  background-size: contain;
  border: 1px solid red;
}
Inténtalo tú mismo

2. Si la propiedad background-size se establece en "100% 100%", la imagen de fondo se estirará para cubrir toda el área de contenido:

Aquí está el código CSS:

Ejemplo

div {
  width: 100%;
  height: 400px;
  background-image: url('img_flowers.jpg');
  background-size: 100% 100%;
  border: 1px solid red;
}
Inténtalo tú mismo

3. Si la propiedad background-size está configurada como "cover", la imagen de fondo se escalará para cubrir toda el área de contenido. Tenga en cuenta que el valor "cover" mantiene la relación de aspecto y es posible que se recorte alguna parte de la imagen de fondo:

Aquí está el código CSS:

Ejemplo

div {
  width: 100%;
  height: 400px;
  background-image: url('img_flowers.jpg');
  background-size: cover;
  border: 1px solid red;
}
Inténtalo tú mismo

Diferentes imágenes para diferentes dispositivos

Una imagen grande puede ser perfecta en una pantalla de computadora grande, pero inútil en un dispositivo pequeño. ¿Por qué cargar una imagen grande cuando de todas formas tienes que reducir su tamaño? Para reducir la carga, o por cualquier otro motivo, puedes usar consultas de medios para mostrar diferentes imágenes en diferentes dispositivos.

Aquí hay una imagen grande y una imagen más pequeña que se mostrarán en diferentes dispositivos:

Ejemplo

/* For width smaller than 400px: */
body {
  background-image: url('img_smallflower.jpg');
}

/* For width 400px and larger: */
@media only screen and (min-width: 400px) {
  body {
    background-image: url('img_flowers.jpg');
  }
}
Inténtalo tú mismo

El elemento HTML <picture>

El elemento <picture> HTML brinda a los desarrolladores web más flexibilidad para especificar recursos de imágenes.

El uso más común del elemento <picture> será para imágenes utilizadas en diseños adaptables. En lugar de tener una imagen que se amplía o reduce en función del ancho de la ventana gráfica, se pueden diseñar varias imágenes para que llenen mejor la ventana gráfica del navegador.

El elemento <picture> funciona de manera similar a los elementos <video> y <audio>. Se configuran diferentes fuentes y la primera fuente que se ajusta a las preferencias es la que se utiliza:

Ejemplo

<picture>
  <source srcset="img_smallflower.jpg" media="(max-width: 400px)">
  <source srcset="img_flowers.jpg">
  <img src="img_flowers.jpg" alt="Flowers">
</picture>
Inténtalo tú mismo

El atributo srcset es obligatorio y define la fuente de la imagen.

El atributo media es opcional y acepta las consultas de medios que encontrará en la regla CSS @media

También debe definir un elemento <img> para los navegadores que no admiten <picture>.
